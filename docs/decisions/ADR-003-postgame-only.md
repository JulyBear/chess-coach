# ADR-003：当前阶段不做对局中提示，产品主线是赛后复盘与训练

## 状态
已采纳

## 背景
用户已经明确项目定位：
- 不做对局中的实时提示、推荐或辅助
- 先从赛后复盘切入
- 后续创新重点放在训练、知识掌握、弱点强化，而不是实时观战式助手

这会直接影响系统边界、产品复杂度和技术实现方式。

## 决策
当前阶段产品不提供对局中的实时提示。

产品主线分两层推进：
1. 先做赛后复盘底座
2. 再扩展为 AI 驱动的训练与强化学习系统

## 原因
- 赛后复盘边界清晰
- 可避免实时系统复杂度
- 可避免 UI 与后端被“实时推荐”需求牵着走
- 训练系统更符合产品长期创新方向
- 更容易形成用户长期学习闭环，而不是一次性看报告

## 放弃的方案
- 实时引擎提示
- 对局中弹出推荐走法
- 实时 AI 解说
- 把产品做成实时外挂式辅助工具

## 影响
- 分析接口可以按“离线触发”设计。
- 前端先以回放、评分、报告展示为主。
- 后续新增功能优先服务复盘体验、题库训练、布局学习、弱点强化。
- 后续创新重点放在：
  - 杀法题库与训练
  - 定式/布局学习
  - 用户个人错误模式总结
  - 个性化强化学习闭环
