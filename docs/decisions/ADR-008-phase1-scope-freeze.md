# ADR-008：第一期范围冻结为复盘展现底座，不提前实现训练系统

## 状态
已采纳

## 背景
为了避免第一期在未来训练需求推动下持续膨胀，需要明确冻结第一期范围。

用户已明确：后续训练能力必须建立在第一期展现底座之上，但这不意味着第一期就要提前把训练系统做完。

## 决策
第一期范围冻结为：
- 采谱链路稳定
- 数据模型收口
- REST API 协议稳定
- 统一复盘展现底座成型
- 文档与决策同步

第一期明确不实现完整训练系统、布局学习系统、实时辅助系统和过度工程化基础设施。

## 原因
- 符合 KISS / YAGNI
- 避免第一期变成半成品大杂烩
- 先把复盘与展现底座做稳，后面训练能力才有统一基础

## 放弃的方案
- 一期内提前实现题库、掌握度、复习系统
- 一期内同时做复盘、训练、布局、强化全套闭环
- 一期为未来需求提前铺设过多复杂基础设施

## 影响
- 一期代码改造优先顺序固定为：`server/db.py` -> `server/main.py` -> `web/review.html`
- 后续训练相关内容只做预留，不进入一期实现范围
- 新增需求若超出冻结清单，应自动归入后续阶段而非塞回一期
