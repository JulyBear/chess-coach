# ADR-002：分析、棋盘、AI 解说采用胶水集成

## 状态
已采纳

## 背景
项目已经引入或适合引入现成能力：
- Pikafish：引擎分析
- Claude：中文复盘解说
- xiangqiboardjs：棋盘展示

这些都不是本项目需要重新发明的部分。

## 决策
分析、棋盘展示、AI 教练报告全部采用胶水式集成，项目自身只负责最小必要的数据转换和流程编排。

## 原因
- 符合 KISS / YAGNI
- 降低维护成本
- 避免把项目做成新的象棋平台

## 放弃的方案
- 自写规则引擎
- 自写棋盘组件
- 自写分析器或复杂多层 AI 流程

## 影响
- `server/engine.py` 保持为 Pikafish 薄封装。
- `server/llm.py` 保持为 Claude 薄封装。
- `server/xiangqi.py` 只保留最小胶水，不扩成完整规则系统。
