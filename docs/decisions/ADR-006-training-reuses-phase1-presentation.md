# ADR-006：后续训练系统必须复用第一期统一展现底座

## 状态
已采纳

## 背景
用户明确要求：后面所有训练都要建立在第一期的展现系统基础上。

这意味着第一期不能只是临时做一个复盘页，而要把它设计成后续题库训练、布局学习、个性化强化都能复用的统一棋盘展现底座。

## 决策
后续训练系统必须复用第一期的统一展现能力，不另起一套棋盘与分析展示体系。

## 统一复用的展现能力
- 棋盘展示
- 走法列表联动
- 当前局面信息
- 引擎评分展示
- 主变展示
- 局势曲线
- AI 解释展示

## 原因
- 避免复盘页、题库页、布局页、强化训练页各做一套展示协议
- 保证后续新增训练功能只是更换数据源，而不是重做前端底层
- 有利于形成统一产品体验

## 放弃的方案
- 复盘单独一套展示逻辑
- 训练再新建第二套棋盘与讲解协议
- 后续每种训练模式独立实现展示层

## 影响
- 第一期前端设计必须按“统一展现底座”思路整理。
- 后续题库、布局、弱点强化的页面，都优先复用第一期的棋盘与分析组件。
- API 协议未来也应尽量围绕统一的局面展示模型演进。
